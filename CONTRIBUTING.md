# Contributing to HaxCord

Thanks for your interest in contributing to **HaxCord**! To keep the project organized, safe, and high-quality, please follow these guidelines.

---

## 🛠 What You Can Contribute

- ✅ Bug reports and feature requests.
- ✅ Code improvements, mods, or plugin enhancements.
- ✅ Documentation updates, tutorials, or examples.
- ✅ Themes, UI tweaks, or design improvements.

---

## ❌ Contribution Limits

To ensure HaxCord remains stable and maintainable:

1. **No Malicious Code** – Do **not** submit exploits, malware, or scripts intended to harm other users or bypass platform protections.  
2. **No Spam or Low-Quality PRs** – Avoid repetitive, trivial, or irrelevant contributions. Every PR should have a clear purpose.  
3. **Scope Limits** – Contributions should be related to HaxCord functionality, mods, themes, plugins, or documentation. General coding experiments unrelated to HaxCord will be closed.  
4. **Size Limits** – Very large or complex changes (e.g., rewriting major systems) require prior discussion in a Discussion or Issue first.  
5. **Respect Licensing** – Only submit code, assets, or resources that you have rights to share.

---

## 📋 How to Contribute

1. **Fork the repository** and create a branch for your changes.  
2. **Test your changes** thoroughly before submitting a PR.  
3. **Submit a Pull Request** with a clear description of your contribution.  
4. **Engage in Discussions** if maintainers request changes or feedback.  

---

By following these guidelines, you help keep HaxCord safe, stable, and enjoyable for the entire community.  

Thank you for contributing! 💜
