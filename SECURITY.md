# Security Policy

## Supported Versions

The following versions of HaxCord are currently supported with security updates and bug fixes.

| Version | Supported |
| v0.2.0 | :white_check_mark: |
| Latest (main branch) | ✅ |
| Pre-release / beta builds | ⚠️ Limited support |
| Older archived versions | ❌ |

We strongly recommend using the **latest version of HaxCord** to ensure you receive security fixes and improvements.

---

## Reporting a Vulnerability

If you discover a security vulnerability in HaxCord, please report it responsibly.

**Do not open a public issue for security vulnerabilities.**

Instead, report it privately by:

- Opening a **GitHub Security Advisory** on this repository
- Contacting the maintainer through GitHub

When reporting a vulnerability, please include:

- A clear description of the issue
- Steps to reproduce the vulnerability
- Potential impact of the vulnerability
- Screenshots, logs, or proof-of-concept code if applicable

---

## What to Expect

After reporting a vulnerability:

1. The maintainer will review the report.
2. You may be contacted for additional information.
3. If confirmed, the vulnerability will be patched in the next update.
4. A public security notice may be released once a fix is available.

---

## Responsible Disclosure

Please allow time for the issue to be resolved before publicly disclosing details of the vulnerability. Responsible disclosure helps protect users of HaxCord.

Thank you for helping keep HaxCord secure.
